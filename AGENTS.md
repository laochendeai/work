## Repo hygiene

- 不要在仓库根目录随手生成临时脚本/报告/备份文件。
- 一次性实验脚本统一放在 `tools/scratch/`（该目录应保持在 `.gitignore` 中）。
- 不要生成或提交 AI 工具目录（如 `.claude/`, `.codex/`, `.cursor/`）；这些应仅作为本地配置并被 git 忽略。
- 不要输出 `*.backup_*` / `*.bak_*` / `*.old_*` 这类“备份文件”，用 git 历史替代。

## 目录约定

- `core/`：主流程代码（爬取/抽取/发送/队列）。
- `tools/`：维护工具（回填、重抽取、评估、数据源管理）。
- `scripts/`：演示/手动运行脚本（不属于主流程）。
- `data/`、`logs/`：运行期产物（应被 git 忽略）。
