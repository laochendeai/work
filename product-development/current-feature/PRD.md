# 产品需求文档 (PRD) - 个人智能设计营销自动化系统

## 1. 项目概述

### 1.1 文档信息
- **文档版本**: v1.0
- **创建日期**: 2025-12-04
- **最后更新**: 2025-12-04
- **文档负责人**: 智能化设计工程师（个人使用）

### 1.2 功能概述
这是一个专为**个人智能化设计工程师**打造的自动化营销工具，能够7*24小时自动搜索政府采购中标公告，提取中标公司联系人信息，并通过自动化方式向潜在客户推荐个人设计团队，帮助获得更多招投标方案设计和施工方案设计项目机会。

### 1.3 目标用户
- **智能化设计工程师（本人）**: 专门从事招投标方案设计、施工方案设计的个人工程师，希望自动化营销推广自己的设计服务

### 1.4 业务目标
- **项目获取**: 每月获得5-10个新的设计项目询盘
- **客户拓展**: 自动建立200+个潜在客户联系信息数据库
- **成本控制**: 总投入控制在500元以内
- **时间效率**: 7*24小时自动运行，每天节省4小时人工搜索时间

## 2. 问题陈述

### 2.1 个人痛点
1. **人工搜索效率低**: 每天花4小时手动搜索政府采购中标公告，效率低下
2. **信息提取困难**: 中标公告中的联系人信息分散，手动提取耗时且容易遗漏
3. **客户跟进不及时**: 发现潜在客户后，缺乏系统化的跟进管理
4. **营销推广范围有限**: 个人精力有限，无法同时跟进大量潜在客户
5. **收入来源不稳定**: 缺乏持续的项目获取渠道，收入波动大

### 2.2 个人机会
智能化设计市场需求旺盛，政府采购项目需要专业的设计团队。通过自动化营销工具可以：
- 建立持续的项目获取渠道
- 扩大个人设计服务的市场覆盖
- 提高时间利用效率，专注于核心设计工作
- 降低营销成本，提高投入产出比

### 2.3 解决方案概述
为个人工程师打造一个轻量化、低成本的自动化营销系统：
- 自动搜索政府采购中标公告，识别智能化相关项目
- 智能提取中标公司联系人信息（姓名、电话、邮箱）
- 自动发送个性化推荐邮件，介绍个人设计团队优势
- 建立潜在客户数据库，便于后续跟进管理

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 智能化项目自动搜索
- **功能描述**:
  - **全面覆盖**: 搜索政府采购网、高校采购平台、国企采购平台、上市公司采购平台
  - **关键词智能匹配**: "智能化"、"弱电"、"安防"、"监控"、"综合布线"、"网络设备"、"智能建筑"、"智慧校园"等
  - **多领域覆盖**:
    - 政府机构：政府部门、事业单位、公共机构项目
    - 教育领域：大学、职业院校、教育机构项目
    - 国企央企：能源、电力、通信、建筑等央企项目
    - 大型企业：互联网、房地产、制造业、金融等行业项目
  - **智能运行**: 7*24小时自动运行，分时段轮询不同网站
  - **项目规模筛选**: 重点筛选10万-1000万的智能化项目，优先50万-300万

- **个人价值**:
  - 覆盖全行业项目机会，不局限于政府项目
  - 建立多元化的客户来源渠道
  - 获得更多高质量的项目询盘

- **验收标准**:
  - 每日发现15-30个相关项目（覆盖所有来源）
  - 项目相关性判断准确率>80%
  - 系统运行稳定率>99%
  - 多源数据整合准确率>95%

#### 3.1.2 联系信息智能提取
- **功能描述**:
  - 自动识别中标公告中的联系人姓名、电话、邮箱
  - 支持多种格式：项目经理、负责人、联系人等
  - 手机号和座机号码都能识别
  - 邮箱格式验证和有效性检查

- **个人价值**: 省去手动查找联系人信息的时间

- **验收标准**:
  - 联系人信息提取成功率>70%
  - 电话号码识别准确率>90%
  - 邮箱地址识别准确率>95%

#### 3.1.3 自动化邮件推荐
- **功能描述**:
  - 根据项目特点自动生成个性化推荐邮件
  - 邮件模板包含：个人简介、专业资质、类似项目经验、联系方式
  - 智能控制发送频率，避免被标记为垃圾邮件
  - 支持邮件跟踪：发送状态、打开率、回复情况

- **个人价值**: 主动向潜在客户展示专业能力，获得项目机会

- **验收标准**:
  - 每日发送10-20封推荐邮件
  - 邮件送达率>95%
  - 垃圾邮件标记率<5%
  - 每月获得5-10个回复询盘

#### 3.1.4 客户数据库管理
- **功能描述**:
  - 本地SQLite数据库存储所有联系信息
  - 记录发送历史和回复情况
  - 客户分级管理：已回复、已联系、待跟进
  - 数据导出功能，支持Excel格式

- **个人价值**: 建立个人客户资源库，便于长期跟进

- **验收标准**:
  - 数据存储安全性100%
  - 查询响应时间<3秒
  - 支持多条件筛选查询

### 3.2 辅助功能

#### 3.2.1 运行状态监控
- 简单的桌面界面显示运行状态
- 每日运行报告：搜索数量、发送数量、回复数量

### 6.1 个人数据库结构（SQLite）
```sql
-- 项目信息表
CREATE TABLE projects (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,           -- 公告标题
  project_name TEXT,             -- 项目名称
  winning_company TEXT,          -- 中标单位
  project_amount REAL,           -- 项目金额
  announcement_date DATE,        -- 公告日期
  source_url TEXT,               -- 信息来源链接
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 联系人信息表
CREATE TABLE contacts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  project_id INTEGER,            -- 关联项目ID
  contact_name TEXT NOT NULL,    -- 联系人姓名
  contact_phone TEXT,            -- 联系电话
  contact_email TEXT,            -- 联系邮箱
  contact_position TEXT,         -- 联系人职位
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 邮件发送记录表
CREATE TABLE email_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  contact_id INTEGER,            -- 关联联系人ID
  email_subject TEXT,            -- 邮件主题
  email_content TEXT,            -- 邮件内容
  send_time TIMESTAMP,           -- 发送时间
  send_status TEXT,              -- 发送状态：成功/失败
  is_replied BOOLEAN DEFAULT FALSE,  -- 是否已回复
  reply_time TIMESTAMP,          -- 回复时间
  FOREIGN KEY (contact_id) REFERENCES contacts(id)
);
```

## 8. 个人使用风险与应对

### 8.1 技术风险（个人版）
- **网站反爬**: 政府网站可能限制访问频率
  - **应对**: 降低访问频率，使用延时，模拟正常用户行为
- **网站结构变化**: 页面改版影响信息提取
  - **应对**: 定期检查提取效果，及时调整解析规则
- **联系信息过时**: 部分联系人可能已更换
  - **应对**: 优先联系最新的公告，定期验证有效性

### 8.2 营销风险
- **邮件被标记为垃圾邮件**: 批量发送可能被邮件服务商限制
  - **应对**: 控制发送频率，个性化邮件内容，使用个人邮箱
- **客户反感**: 部分收件人可能不欢迎自动营销
  - **应对**: 邮件内容专业礼貌，提供真实价值，提供退订选项
- **竞争激烈**: 智能化设计市场竞争可能比较激烈
