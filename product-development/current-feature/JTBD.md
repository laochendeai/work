# Jobs to be Done - 智能政府采购信息收集与联系管理

## 核心Jobs to be Done

### JTBD 1: "帮助我及时发现和获取相关的政府采购商机"

**用户情境**: 作为弱电智能化公司的商务开发人员，我需要持续监控政府采购市场，确保不会错过任何潜在的商业机会。

**当前挣扎**:
- 政府采购网站众多，信息分散，需要手动访问多个网站
- 关键词搜索结果不准确，包含大量无关信息
- 信息更新不及时，容易错过重要项目的投标截止时间
- 人工筛选耗时耗力，效率低下

**期望结果**: 能够自动、准确、及时地获取与我的业务相关的政府采购项目信息，提高商机发现的效率和准确性。

### JTBD 2: "帮我快速识别和提取项目中的关键联系信息"

**用户情境**: 找到感兴趣的政府采购项目后，我需要快速找到项目联系人、电话、邮箱等关键信息，以便及时进行商务沟通。

**当前挣扎**:
- 政府采购公告中的联系信息格式不统一，难以快速定位
- 联系信息分散在公告的不同位置，需要仔细阅读整个文档
- 部分公告缺少关键联系信息，浪费跟进时间
- 手动提取容易出错，影响沟通效率

**期望结果**: 能够自动识别和提取项目中的完整联系信息，并以结构化的方式呈现，方便我快速进行商务联系。

### JTBD 3: "帮我系统化管理项目信息和客户关系"

**用户情境**: 在跟进多个政府采购项目时，我需要系统化管理项目信息、联系记录、跟进计划等，确保没有遗漏任何重要的商机。

**当前挣扎**:
- 项目信息分散在不同的Excel表格和文档中，难以统一管理
- 缺乏系统性的跟进计划，容易忘记重要的跟进节点
- 团队成员之间信息不共享，协作效率低
- 历史项目数据难以利用，无法进行有效的经验总结

**期望结果**: 建立统一的项目管理和客户关系管理系统，实现信息的集中存储、团队协作和智能化的跟进提醒。

### JTBD 4: "帮助我分析市场趋势和竞争情况"

**用户情境**: 作为市场分析师或销售经理，我需要了解政府采购市场的整体趋势、竞争对手的表现、以及我公司的市场地位。

**当前挣扎**:
- 缺乏系统性的市场数据收集和分析工具
- 竞争对手信息难以获取，无法了解其项目类型和报价水平
- 无法准确评估项目的商业价值和竞争激烈程度
- 缺乏数据支持，难以制定有效的市场策略

**期望结果**: 获得基于真实数据的市场分析报告，了解行业趋势、竞争格局和项目价值，为业务决策提供数据支持。

## 次要Jobs to be Done

- 分析历史中标项目的特征和规律
- 提供投标策略建议和风险评估
- 跟踪投标进度和结果反馈

### JTBD 6: "帮助我节省时间和成本"
- 减少人工信息收集的时间投入
- 降低信息获取和处理的成本
- 提高整体工作效率

### JTBD 7: "帮助我与团队协作"
- 共享重要的项目信息和联系资料
- 分配跟进任务和跟踪执行情况
- 建立团队知识库和经验分享
