# 代码质量审查

执行全面的代码质量审查：$ARGUMENTS

## 当前状态

- Git状态: !`git status --porcelain`
- 最近更改: !`git diff --stat HEAD~5`
- 仓库信息: !`git log --oneline -5`
- 构建状态: !`npm run build --dry-run 2>/dev/null || echo "无构建脚本"`

## 任务

按照以下步骤进行彻底的代码审查：

1. **仓库分析**
   - 检查仓库结构，识别主要语言/框架
   - 查找配置文件（package.json、requirements.txt、Cargo.toml等）
   - 审查README和文档以了解上下文

2. **代码质量评估**
   - 扫描代码异味、反模式和潜在错误
   - 检查一致的编码风格和命名约定
   - 识别未使用的导入、变量或死代码
   - 审查错误处理和日志记录实践

3. **安全性审查**
   - 查找常见的安全漏洞（SQL注入、XSS等）
   - 检查硬编码的密钥、API密钥或密码
   - 审查身份验证和授权逻辑
   - 检查输入验证和清理

4. **性能分析**
   - 识别潜在的性能瓶颈
   - 检查低效的算法或数据库查询
   - 审查内存使用模式和潜在泄漏
   - 分析包大小和优化机会

5. **架构与设计**
   - 评估代码组织和关注点分离
   - 检查适当的抽象和模块化
   - 审查依赖管理和耦合
   - 评估可扩展性和可维护性

6. **测试覆盖率**
   - 检查现有测试覆盖率和质量
   - 识别缺乏适当测试的区域
   - 审查测试结构和组织
   - 建议额外的测试场景

7. **文档审查**
   - 评估代码注释和内联文档
   - 检查API文档的完整性
   - 审查README和设置说明
   - 识别需要更好文档的区域

8. **建议**
   - 按严重程度优先级排序问题（关键、高、中、低）
   - 提供具体、可操作的建议
   - 建议改进的工具和实践
   - 创建包含后续步骤的摘要报告

请记住要具有建设性，并在适用的地方提供具体示例，包括文件路径和行号。
